General

Defines directories for raw images (Image folder) and output Analysis folder. Channels, wells (position) and time frames can be constrained here.
Settings are saved to a *.conf file (simple text file).

Parameters

Image folder

Contains raw images as saved by the microscope. Cellcogition supports TIFF stacks (one image per file), structured similar to our demo data, which must have

  • one subdirectory per movie (referred to as "position"), and
  • similar naming schema - named tokens in the filename).

Multipage TIFFs are interpreted as z-stacks, but the the corresponding token is then ignored. TIFFs must be encoded either in UINT8 or UINT16 (see 16 to 8 bit conversion in Object Detection). LZW or ZIP compressed TIFFs are supported.

Analysis folder

All results are save in the Analysis folder except classifier annotation and training data.

The subdirectory structure is:

Naming scheme


Image files must follow a naming schema in order to be imported.
So far, only image sequences with token in the filename are supported (see Data folder).
Schema definitions can be added to a the file naming_schemes.conf, which is located in the resources sub-folder of this application (for MacOS: it's part of the .app package).
The possibility to add/edit schemas from the GUI or to import data from OME / BioFormats will be added in the future.








Constrain positions


Turn position constraint on or off. If positions are specified defines which positions of the experiment are processed. 





Positions


List of position names separated by comma. Each name can contain letters, digits or the underscore.





Skip processed positions


Defines if already processed positions should be skipped. An empty file is created for every finished position in log/_finished/ of the output folder.








Constrain timepoints


Turn timepoint constrains on or off. Defines which frames (time points) of the experiment are processed.





First


The first frame to be processed. If this value is outside the possible range the minimal possible frame will be used.





Last


The last frame to be processed. If this value is outside the possible range or smaller than the first value the maximal possible frame will be used.





Timepoint increment


The step-size in frames from the first to the last frame.





Image cropping


Specify a sub-rectangle of the image data for analysis.







Control Panel



Load or save parameters to a .conf file.
