Error Correction

Sequences of class labels as obtained by the event selection can be corrected for errors by incorporation of time information.

Correction of errors is performed by a hidden Markov model (HMM), which is learned from all event-sequences of one experimental condition.
The HMM is implemented in scripts of the R-project and requires installation of additional packages (this is going to change soon). CecogAnalyzer calls the R script, which reads the event-based trajectory files and generates the HMMs. The visualized results are written to one HTML page per channel, which opens automatically after processing is completed. The user can constrain graphs by specifying one text file per channel. The fully connected graph implying no constrains is used per default. The association between positions (individual movies) and experimental information such as oligo identifier or the name of the targeted gene symbol can also be defined in a simple text file (see example files in the generated hmm folder). Based on this information trajectories are grouped by position, oligo ID or gene symbol and HMMs are generated per group.
Error correction can be tested separately for already picked events, e.g. by motif selection in Tracking.

Parameters

R-project executable

Filename of the R executable. If the R-project is not part of the system's PATH variable the full filename is required, e.g. C:\Program Files\R\R-2.10.0\bin\R.exe on Windows for R 2.10.0.

Constrain graph

A priori information about a cellular process can be applied by constraining the transition between cellular states. These transitions can be expressed as a graph structure and per default the fully connected graph (all connections between states are possible) is used. The transition probabilities of the HMM are learned according to the underlying graph (the probability is zero for all missing edges).
If this option is omitted the fully connected graph is used and written as text file to the hmm folder (which can be modified and used as example).

Primary file

Text file defining the graph structure for the classifier of the primary channel./p>

Secondary file

Text file defining the graph structure for the classifier of the secondary channel.

Position labels

Association of the experimental condition to a position (also named plate mapping or plate layout). For now only oligo ID and gene symbol are supported. If omitted the position names are used for identification of the experiment and a sample file <OUTPUT>/hmm/layout.txt is created.

File

Simple tab-separated text file with three columns: Position (must match the position names), OligoID (any string), and GeneSymbol (any string).

Group by

The HMM is learned and errors are corrected per experimental condition. How positions are grouped can be defined here based on the position labels above.

Position

Group by Position (no position labels necessary).

Oligo ID

Group by oligo ID (defined in the position labels).

Gene symbol

Group by gene symbol (defined in the position labels).

Plot parameter

Parameters for plot generation.

Time-lapse [min]

Specifies the time-lapse.

Max. time in plot [min]

The maximal time (y-axis) in a bar plot or box plot.

Control Panel

  • Correct errors - Performs HMM error correction on event trajectories already present in the output sub-folders (e.g. generated either by Processing or Apply event selection in Tracking. Error correction is only performed for primary and/or secondary channel if the corresponding classification is enabled in Processing.
